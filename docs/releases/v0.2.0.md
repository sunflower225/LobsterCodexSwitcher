# LobsterCodexSwitcher v0.2.0

This release establishes the public `LobsterCodexSwitcher` identity and ships the first Lobster/OpenClaw-specific proxy automation flow.

## Highlights

- Rebrands the project as `LobsterCodexSwitcher`
- Keeps the existing `codex-switcher` / `csw` command names for compatibility
- Synchronizes `~/.cli-proxy-api` whenever you switch Codex accounts
- Restarts local `cliproxyapi` automatically after proxy sync
- Triggers proxy OAuth login automatically when the target account does not have a local proxy credential yet

## Verification

- `python3 -m unittest tests/test_proxy_sync.py`
- `python3 -m py_compile codex_switcher.py codex-switcher.py install.py`

## Recommended usage

```bash
codex-switcher --switch flow14662@gmail.com --json
```

The command now switches both the active Codex auth and the local proxy auth path in one step.
